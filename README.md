 **Генерация всех путей в лабиринте 5x5**
Задание

Вариант 15: Написать программу для генерации всех путей в лабиринте размером 5×5 из левого верхнего угла (0,0) в правый нижний угол (4,4).

__________________________________________________________
 **Алгоритм**

Шаг 1: Инициализация

· Создать пустой список для хранения всех найденных путей
· Начать с начальной позиции (0,0)
· Добавить начальную позицию в текущий путь

Шаг 2: Проверка достижения цели

· ЕСЛИ текущая позиция = конечная позиция (4,4):
  · Сохранить копию текущего пути в список всех путей
  · ВЕРНУТЬСЯ на предыдущий уровень рекурсии

Шаг 3: Попытка движения ВПРАВО

· ЕСЛИ справа есть клетка И она свободна (не стена):
  · Перейти в правую клетку
  · Добавить новую позицию в текущий путь
  · Вызвать алгоритм рекурсивно из новой позиции
  · УДАЛИТЬ последнюю позицию из пути (backtrack)

Шаг 4: Попытка движения ВНИЗ

· ЕСЛИ снизу есть клетка И она свободна (не стена):
  · Перейти в нижнюю клетку
  · Добавить новую позицию в текущий путь
  · Вызвать алгоритм рекурсивно из новой позиции
  · УДАЛИТЬ последнюю позицию из пути (backtrack)

Шаг 5: Завершение

· Когда все возможные направления из текущей позиции проверены
· Автоматически возврат на предыдущий уровень (backtrack)

Процесс Backtracking

Механизм возврата:

1. После рекурсивного вызова УДАЛЯЕМ последнюю позицию
2. Это позволяет "откатиться" и попробовать другие пути
3. Каждый вызов функции имеет свой контекст пути

**Условия остановки**

Успех: Достигнута позиция (4,4)
Тупик:Нет свободных клеток справа и снизу
Завершение:Все возможные пути исследованы


**Временная сложность**

В худшем случае: O(2^(n+m))
Где n, m - размеры лабиринта (5×5)
На практике сложность меньше из-за наличия стен

 **Контрольный вопрос**

Опишите стратегию «разделяй и властвуй» на примере алгоритма сортировки слиянием.

Стратегия «разделяй и властвуй» — парадигма разработки алгоритмов, состоящая из трех этапов:

1. Разделить (Divide): Исходная задача разбивается на меньшие подзадачи
2. Властвовать (Conquer): Подзадачи решаются рекурсивно
3. Комбинировать (Combine): Решения подзадач объединяются

Пример - сортировка слиянием:

· Разделить: Массив рекурсивно делится пополам
· Властвовать: Подмассивы из 1 элемента считаются отсортированными
· Комбинировать: Отсортированные подмассивы сливаются в один

Сложность: O(n log n)
